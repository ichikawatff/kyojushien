# 葛飾区特化 居住支援法人サイト 仕様書（静的サイト／バイブコーディング用）

## 1. プロジェクト概要
### 目的
葛飾区に特化した居住支援法人の公式サイトを制作し、支援内容の説明・相談導線・物件情報の掲載を行う。対象は「住まいに困っている方」「支援機関・行政」「家主・管理会社」。信頼性と分かりやすさを最優先とする。

### ゴール
- **ローカルで動作**：`index.html` をダブルクリックして開ける（サーバ不要）。
- **相談導線が明確**：電話／Webフォームへの誘導が明確。
- **物件掲載が可能**：供給予定の賃貸物件を掲載・更新しやすい構造。

## 2. 対象ユーザーと想定ニーズ
### 主なターゲット
- 住まいに困っている方（住宅確保要配慮者の可能性がある方）
  - 何ができるか、費用、必要書類、流れ、どこに連絡すればいいか。
- 支援機関・行政・相談窓口
  - 連携先としての信頼性、実施内容、対応範囲、実績（今後追加）。
- 家主・管理会社
  - 受け入れ不安の解消（滞納・トラブル時の支援、見守り等）。
  - 物件供給の協力依頼。

## 3. 要件（必須／推奨）
### 必須要件
- **完全静的**：HTML/CSS/JSのみ。ビルドツール不要。
- **ローカルで動作**：`file://` でも崩れない（相対パス運用、`fetch`制限に配慮）。
- **レスポンシブ対応**：スマホ最優先。
- **葛飾区特化を明示**：地名・地図・対応エリアを明確に。
- **物件掲載枠**：自社供給予定の賃貸物件を掲載できるページを用意。

### 推奨要件
- 物件一覧は **JSON相当のJS配列**から自動生成。
- 問い合わせは **静的運用向け**（Googleフォーム埋め込み or mailto）。
- アクセシビリティ：読みやすい文字サイズ・コントラスト・キーボード操作。

## 4. 技術仕様（ローカル前提）
### 技術スタック
- HTML5 / CSS3 / Vanilla JavaScript
- フォント：システムフォント優先
- 画像：`assets/img/`（WebP推奨）

### ローカル制約への対応
`file://` では `fetch()` が制限される可能性があるため、**物件データはJS変数として同梱**する。

- **方式A（採用）**：`assets/js/listings.js` に `LISTINGS` 配列を定義（`fetch`不要）。
- 方式B（任意）：ローカルサーバ起動（`python -m http.server`）をREADMEに記載。

## 5. サイト構成（ページ一覧）
### 共通（全ページ）
- **ヘッダー**：ロゴ／サイト名、ナビ、CTA
- **フッター**：法人情報、住所、対応エリア、電話、受付時間、プライバシーポリシーリンク
- **CTA**：
  - メイン：`Webフォームで相談する`
  - 補助：電話／メール

### ページ1：トップ（index.html）
- 見出し例：「葛飾区の住まいの困りごと、相談できます」
- サブ：対象例（高齢・障害・ひとり親・DV・生活困窮など配慮表現）
- CTAボタン（フォーム／電話）
- できること（3〜6項目）
- 支援の流れ（1→2→3）
- 対応エリア（葛飾区中心・近隣は要相談）
- 最新物件（3件カード→一覧へ）
- お知らせ（任意）

### ページ2：居住支援について（support.html）
- 支援メニュー：入居相談、申込支援、見守り・定期連絡、生活相談連携、調整
- 対象となりやすい方（断定は避ける）
- 費用（未定なら「個別にご案内」）
- 必要書類（状況により異なる）

### ページ3：物件情報（listings.html）
- 絞り込み（家賃帯／間取り／駅／即入居／条件）
- 物件カード一覧（画像、家賃、管理費、間取り、所在地、最寄駅、特徴）
- 詳細ページへ（`listing.html?id=xxx`）
- 注意書き（空室変動・内見方法・申込条件）

### ページ4：物件詳細（listing.html）
- 画像ギャラリー（最低1枚）
- 基本情報（家賃、敷金礼金、間取り、面積、築年、構造、所在地、交通）
- 設備・条件（事実ベース）
- 地図（Google Mapリンク or 画像）
- CTA：物件問い合わせ（フォームに物件ID自動入力）

### ページ5：相談・お問い合わせ（contact.html）
- **Webフォームが主導線**であることを明記
- 連絡手段：電話／メール／フォーム
- フォーム項目：
  - 氏名（任意）
  - 連絡先（必須：電話 or メール）
  - 相談内容（必須）
  - 希望条件（任意）
  - 物件ID（任意・自動入力）
- 送信方式（推奨順）：
  1. Googleフォーム埋め込み（最推奨）
  2. 外部フォーム（Formspree等）
  3. `mailto:` 下書き生成

### ページ6：法人情報（about.html）
- 法人概要（名称、所在地、連絡先、営業時間）
- ミッション（葛飾区で住まいの安心を支える）
- 連携方針（行政・支援機関・家主）
- 採用・協力募集（任意）

### ページ7：FAQ（faq.html）
- よくある不安に回答（費用、保証人、緊急連絡先、生活保護、外国籍 等）
- “ケースにより異なる”はOKだが、必ず相談導線へ戻す

### ページ8：プライバシーポリシー（privacy.html）
- 個人情報の取り扱い（フォーム／メール／電話の記録等）

## 6. 物件データ仕様（更新が楽な形）
### 方式
`assets/js/listings.js` に `LISTINGS` 配列で定義（`fetch`不要）。

### データ項目（例）
```js
const LISTINGS = [
  {
    id: "kts-0001",
    title: "葛飾区・駅徒歩8分 1K",
    status: "available", // available / coming_soon / reserved
    rent: 58000,
    fee: 3000,
    layout: "1K",
    area_sqm: 22.1,
    address_area: "葛飾区（町名まで）",
    station: "最寄駅 徒歩8分",
    features: ["バス・トイレ別", "室内洗濯機置場", "見守り相談可"],
    notes: "条件は状況によりご相談ください。",
    images: [
      "assets/img/listings/kts-0001/01.webp",
      "assets/img/listings/kts-0001/02.webp"
    ],
    updated_at: "2026-01-20"
  }
];
```

### 表示ルール
- 家賃は `rent` を円表示、未定なら「要相談」も許可。
- `status` によりカードバッジ表示（募集中／準備中／申込あり）。

## 7. デザイン指針（信頼・公共性）
- トーン：やさしい／落ち着き／丁寧
- 色：白ベース＋アクセント1色（緑・青系）
- 写真：葛飾の空気感＋室内写真
- 文字：大きめ、行間広め
- UI：カード＋余白、CTAは目立つが過剰演出なし

## 8. SEO / 表示要件
- title/description を各ページに設定
- OGP設定
- alt属性の徹底
- 画像はWebP・遅延読み込み（`loading="lazy"`）

## 9. 文言の注意
- 断定しすぎない（制度・状況で変わる）
- 医療・法律に踏み込みすぎない
- 差別的表現を避ける（属性ではなく状況ベース）
- 葛飾区特化は強調、周辺は「要相談」
